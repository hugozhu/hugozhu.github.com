用 Agent 在钉钉上跟进团队周度任务

From sending notifications to building feedback loops — tracking 16 executives' weekly progress with an AI Agent

 Posted on July 9, 2026  |  2 min  |  218 words

每周一早上，我在董事群发一条消息，列出 16 位高管本周的重点事项。

接下来一周，我靠记忆和同事的自觉来掌握进度。有人及时反馈，有人到周五才想起有这回事，有人直接忘了。

上周五，我试了一下让 AI Agent 帮我跟。结果超出预期。

问题：谁在跟进，谁没跟上

团队周度任务跟进这件事，本质上是一个 Feedback Loop 问题：管理者派出任务，高管执行并反馈，管理者根据反馈调整。

但在实际操作中，这个环路经常断裂。原因很简单——管理者没有时间去 16 个单聊里一条条翻消息，然后和任务清单逐条比对。你只能等别人来找你，或者在群里 @所有人 催一遍。

这正是我在 人机协同的核心不是 Chat，是 Feedback Loop 里说的那个问题：Chat 模式假设人知道该问什么，但在管理场景下，你不知道该问谁、哪条消息是进展、哪条是闲聊。

上周五下午，我做了一个实验：让 AI Agent 把这件事接管下来。

实操流程

以下是我实际跑通的四步流程。每一步都是真实操作，不是演示。

第一步：发出任务清单

周一早上在董事群发一条包含所有高管任务的消息。这是起点，也是 Agent 后续比对的基准。

任务清单同时写进在线文档。文档是结构化的——每个高管一个章节，列清楚具体任务项。这个文档就是整个闭环的「状态表」。

举个例子，任务清单可能长这样：

### @张总（销售VP）
1. Q3 销量目标拆解，7月30日里程碑
2. 高端品牌发布会方案确认
3. 重点经销商渠道拓展：华东区、华南区

### @李总（研发VP）
1. 新一代混动平台验证，周四首台样车下线
2. 智能座舱 OTA 灰度发布计划
3. 欧洲法规合规车型开发进度

### @王总（生产VP）
1. 合肥工厂二期产线投产准备
2. 核心供应商交付稳定性保障
3. 产能规划与成本优化方案

第二步：Agent 主动去读每个人的单聊

这是 Feedback Loop 成立的关键一步。

不是我让 Agent 去问每个人「你这周任务进展如何」——那是 Chat 模式，人发起，Agent 响应。

而是让 Agent **主动 **去读我和每位高管的单聊消息，从中提取与任务相关的反馈。Agent 逐条扫描消息，判断哪些是在说任务进展，哪些是无关寒暄。

这一步并行跑 16 个人的单聊，几分钟全部拉完。

第三步：自动标记，发给每个人确认

读完单聊后，Agent 对每条任务给出判断：

  • 有相关进展反馈 → 标记 ✅，附上进展摘要
  • 没有反馈 → 标记 ⚠️，列出待跟进项

然后把结果发给每一位高管，消息里只包含他们自己的任务。消息格式是：

本周重点事项跟进：
✅ 新一代混动平台验证 — 首台样车已下线
✅ 智能座舱OTA灰度 — 已立项4个功能包，本周五输出测试报告

两项都已在正轨，继续保持 👍

或者：

本周重点事项跟进提醒：
⚠️ 本周4项均未收到直接反馈，请尽快反馈：
- Q3 销量目标拆解
- 高端品牌发布会方案确认
- 华东区经销商渠道拓展
- 华南区渠道拓展

这一步很重要： 它不是群发通知，而是个性化跟进。每个人的消息内容不同，取决于他们实际的反馈情况。Agent 在第二步读到的消息决定了这一步的输出。

第四步：进度写回文档

反馈汇总追加到在线文档末尾，形成一份本周进度快照。下次再看的时候，上周的进度已经沉淀下来了。

这为什么是一个 Feedback Loop

把这个流程和传统做法对比一下：

传统做法是：管理者发任务 → 高管自己记着 → 管理者想起来去问 → 高管回复 → 管理者拼凑进度。这个环路靠人的主动性和记忆力维持，断裂是常态。

加了 Agent 之后的流程是：管理者发任务 → Agent 主动扫描消息 → Agent 判断进展并个性化跟进 → 高管收到针对性提醒 → 高管给出反馈 → Agent 更新文档。

关键区别在两点：

第一， Agent 是主动的。不是我去问「每个人进展如何」，而是 Agent 自己去读消息、自己判断、自己跟进。这是 Feedback Loop 的核心——Agent 主动行动，而不是等人来问。

第二， 反馈是有状态的。每次跟进的结果都写进了文档。下周再跑的时候，Agent 能看到上周的进度，知道哪些已经完成了，哪些是遗留的。这就是持久记忆的价值——没有记忆，Feedback Loop 是假的。

你可以怎么复用这个流程

这个流程不局限于团队周度任务。它的底层模式是：

1. 有一个任务清单（作为状态基准）
2. Agent 主动从多处收集反馈（单聊、群聊、会议记录）
3. Agent 判断每条任务的状态，个性化跟进
4. 进度持久化到文档（为下一轮闭环提供记忆）

把这个模式套到你的场景里：

  • 如果你是项目总监，任务清单就是项目里程碑，反馈来自各模块负责人的消息
  • 如果你是区域总经理，任务清单就是区域经营指标，反馈来自各省负责人的日常沟通
  • 如果你是销售 VP，任务清单就是客户 pipeline，反馈来自各级销售的单聊

核心操作不变：让 Agent 主动读消息、主动判断、主动跟进、主动沉淀。

你不需要写代码。你只需要：一个能读你日常沟通消息的 AI Agent，一份结构化的任务文档，和每周一次「帮我跟一下进度」的指令。

一个判断标准

怎么知道你做对了？

如果你发现自己还在一个个聊天窗口里翻看进展——你还在 Chat 模式，你是发起者。

如果你发现 Agent 给你推了一份「谁完成了谁没完成」的汇总——你开始构建 Feedback Loop，Agent 开始主动。

如果你发现 Agent 给每个人发了个性化跟进消息，而且文档上周和这周的进度都能对上——Feedback Loop 成立了。

从 Chat 到 Feedback Loop，不是从「用 AI」到「更好地用 AI」。它是从「人驱动 Agent」到「Agent 驱动人做决策」。区别就在那个主动性上。
